What Challenges Did You Face as You Were Improving the Artifact?

    A vector of vectors is an inappropriate data structure for storing objects that represent CSV rows. This is because the application must allow the deletion of rows, which can be problematic. When a row is deleted, all subsequent vectors (representing rows) must be shifted to maintain the structure's continuity. What Lessons Did You Learn From These Challenges?

    Careful system design pays off. It is vital to thoroughly consider the requirements of a data structure before selecting it. Doing so can prevent costly code refactoring and ensure the system meets its performance and functionality goals. Components where the design and pseudocode were thoughtfully planned required the least refactoring and were faster to implement.
